單元測試錯誤排解 (1)

7月 01, 2016

今天踩到一個自己種的雷，使用 res.ok 的時候少了一個 { }，但其實如果你只傳一個物件，物件本身外層本來就有 { } 不需要再多一個 { }，但如果是傳基本型別，就一定得要有 { } 存在，缺少 { } 的時候，單元測試會抓到 res.statusCode == 1，還蠻神奇的。
